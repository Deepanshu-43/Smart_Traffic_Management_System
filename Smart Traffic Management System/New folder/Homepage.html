Core AI Capabilities

At the heart of STMS are powerful AI modules that provide comprehensive, real-time traffic analysis. Each module serves a specific purpose, from initial detection to detailed classification and tracking.

YOLO Object Detection

YOLO (You Only Look Once) provides high-speed, real-time object detection. This chart shows a typical breakdown of objects identified in a busy urban frame.

Vehicle Classification

A specialized module counts and classifies all vehicle types, which is essential for accurate traffic flow analysis and understanding road usage patterns.

Traffic Flow Analysis

The Object Tracking module monitors vehicle movement over time. This allows the system to identify peak hours and quantify traffic volume, as shown in this hourly breakdown.

Logged Event Types

The system logs all significant events for review. While vehicle detection is most common, the identification of "Speed Violations" (a key area for future work) is critical for safety.

Project Roadmap

This 14-week project is structured for success, moving methodically from initial research and system design to full-scale AI integration, testing, and final deployment.

Weeks 1-4: Research & Design

Requirement analysis, literature review, system architecture design, and UI/UX prototyping for the web dashboard.

Weeks 5-9: Backend & AI Integration

Setup of the Python/Flask backend, database integration, and implementation of core AI models (YOLO, PyTorch, Tracking).

Weeks 10-12: Frontend & API Hookup

Development of the HTML/CSS/JS frontend and connecting all dashboard components to the backend API endpoints.

Weeks 13-14: Testing & Deployment

End-to-end system testing, debugging, performance optimization, and preparation of the final report and deployment.
